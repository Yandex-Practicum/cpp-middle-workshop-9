## Начало работы

1. Склонируйте данный репозиторий.
2. Откройте проект в `Visual Studio Code`.
3. Нажмите `F1` и откройте проект в dev-контейнере командой `Dev Containers: Reopen in Container`.

Признаком успешности запуска контейнера, является прохождение всех тестов без ошибок. После того как образ успешно соберется, должен запутиться терминал с ожиданием ввода команды, после этого можно начинать работу.

## Описание инфраструктуры разработки и полезные команды

Будет запущено два контейнера:

1. **Симулятор данных** — эмулирует два сервера, передающих книгу ордеров с ценами на апельсины
2. **Контейнер для разработки** — содержит все необходимые инструменты для разработки и сборки проекта

## Команды для сборки проекта

Для сборки программы используйте один из способов:

**Способ 1: Горячие клавиши**

- `Ctrl+Shift+B` (или `Cmd+Shift+B` на macOS) → выберите тип сборки:
  - `Release` — оптимизированная сборка
  - `Debug` — отладочная сборка

**Способ 2: Через командную палитру**

- Нажмите `F1` → введите `Run Task` → выберите `build`

## Поиск мест для доработки

В коде расставлены метки `#warning`, указывающие на места, требующие вашей доработки:

```cpp
#warning TODO: необходимо реализовать функцию расчёта спреда
```

Чтобы увидеть все необходимые доработки:

1. Соберите проект — компилятор покажет все предупреждения `#warning`
2. Откройте панель "Problems" в VS Code (`Ctrl+Shift+M` / `Cmd+Shift+M`) — там будут отображены все файлы и строки с `#warning`

Также в коде присутствуют комментарии, указывающие, куда именно нужно добавить свой код:

```cpp
#warning "TODO: Параллельно подключаемся к обеим биржам"
/*
    Ваш код здесь: используйте connect_sender для подключения к обеим биржам и получения WebSocket stream'ов.
     - Не забудьте обернуть ioc в std::reference_wrapper при передаче в sender
     - Сохраните полученные WebSocket stream'ы для дальнейшего использования
*/
```

## Подготовка ветви в которой будет вестись разработка

В репозитории организована специальная структура ветвей для пошагового выполнения задач. Ветви расположены последовательно — каждая следующая развивается из предыдущей:

```bash
main → 1_draft → 1_solution → 2_draft → 2_solution → 3_draft → 3_solution
```

Каждая пара `*_draft` → `*_solution` представляет собой один этап работы:

- `1_draft` → `1_solution` — задание 1: подключение и закрытие соединений
- `2_draft` → `2_solution` — задание 2: подписка на каналы
- `3_draft` → `3_solution` — задание 3: парсинг и расчёт спреда

**Порядок работы:**

1. **Создайте свою ветвь для первого задания** — оттолкнитесь от `1_draft`:

```bash
git checkout 1_draft
git checkout -b 1_draft_your_name
```

2. **Реализуйте своё решение** — выполните задачу, описанную в текущем этапе. В коде расставлены `#warning` и комментарии, указывающие места для доработки.

3. **Сделайте коммит** — зафиксируйте свои изменения либо через консоль, либо через IDE

4. **Создайте попытку merge в ветвь `1_solution`** — это позволит увидеть конфликты и сравнить решения:

```bash
git merge 1_solution
```

5. **Разрешите конфликты** — откроется окно разрешения конфликтов, где вы сможете:
   - Сравнить своё решение с авторским
   - Проанализировать различия в подходах
   - Принять авторское решение (ОБЯЗАТЕЛЬНО!)

В данном окне необходимо принять авторское решение "Accept Incoming" для всех конфликтов и затем нажать кнопку "Complete merge" в правом нижнем углу.

Затем необходимо зафиксировать коммит с решением конфликта

6. **Перейдите к следующему заданию** — после разрешения конфликтов смержитесь с веткой `2_draft`:

```bash
git merge 2_draft
```

7. **Повторите процесс** для каждого последующего задания.

**Зачем такая структура:**

- Вы постепенно переходите от простого к сложному
- Можете сравнить своё решение с эталонным
- Видите, как решение развивается от этапа к этапу
- Учитесь работать с конфликтами слияния в контролируемой среде

## Дополнительно

Для настройки автодополнения `Ctrl + Space` нажмите `F1` и выполните команду `clangd: Download language server`. VS Code сам предложит установить подходящую версию clangd (всплывашка в правом нижнем углу). После завершения установки перезагрузите окно кнопкой перезапуска справа снизу или с помощью `F1` и выполните команду `Developer: Reload Window`.
